# 영수증 리뷰 생성기 (SnapReview) MVP PRD

## 핵심 정보

**목적**: 영수증 촬영/업로드만으로 AI가 리뷰 문장을 자동 생성하여, 리뷰 작성에 드는 시간과 노력을 제거한다.
**사용자**: 네이버/카카오/구글 등 플랫폼에 리뷰를 남기고 싶지만 문장 작성이 번거로운 한국인 소비자.

---

## 사용자 여정

```
1. 홈 페이지 (첫 진입)
   ↓ 영수증 업로드 버튼 클릭 or 카메라 촬영 버튼 클릭

2. 업로드 처리 페이지 (로딩)
   ↓ 이미지 수신 → Google Vision OCR 요청 → Gemini AI 리뷰 생성
   ↓ (처리 실패 시) → 홈 페이지로 복귀 + 에러 메시지 표시

3. 리뷰 결과 페이지
   ↓ OCR 추출 정보 확인 + AI 리뷰 3가지 버전 표시

   [리뷰 선택 후 그대로 복사] → 클립보드 복사 완료 → 홈 페이지 or 히스토리 페이지
   [리뷰 수정 원할 시] → 리뷰 에디터 활성화 → 수정 완료 → 클립보드 복사 완료
   [마음에 드는 버전 없을 시] → AI 재생성 요청 → 새 리뷰 3가지 표시

4. 히스토리 페이지
   ↓ 로컬스토리지에서 최근 리뷰 목록 표시

   [과거 리뷰 재사용] → 해당 리뷰 에디터 활성화 → 수정 or 복사
   [히스토리 삭제] → 항목 삭제 확인 → 목록 갱신
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| F001 | 영수증 업로드 | 카메라 촬영 또는 파일 선택으로 영수증 이미지 입력 | 서비스 진입점, 없으면 서비스 불가 | 홈 페이지 |
| F002 | OCR 정보 추출 | Google Vision AI로 상호명/메뉴/금액/날짜 추출 | 리뷰 생성의 핵심 재료 확보 | 업로드 처리 페이지 |
| F003 | AI 리뷰 3버전 생성 | Gemini API로 짧게/보통/상세 3가지 리뷰 자동 생성 | 서비스의 핵심 가치, 리뷰 작성 노력 제거 | 리뷰 결과 페이지 |
| F004 | 리뷰 인라인 에디터 | 생성된 리뷰를 직접 수정할 수 있는 텍스트 에디터 | 사용자 맞춤화 없으면 재방문 동기 없음 | 리뷰 결과 페이지 |
| F005 | 원클릭 클립보드 복사 | 선택한 리뷰를 한 번의 탭/클릭으로 클립보드에 복사 | 서비스 완결점, 복사가 목적 | 리뷰 결과 페이지, 히스토리 페이지 |
| F006 | AI 리뷰 재생성 | 현재 리뷰가 마음에 들지 않을 때 새 버전 3가지 요청 | 사용자 만족도 보장, 1회 생성으로 부족할 수 있음 | 리뷰 결과 페이지 |
| F007 | 리뷰 히스토리 조회 | 로컬스토리지에 저장된 최근 리뷰 목록 표시 및 재사용 | 이전 리뷰 재활용 니즈, 반복 사용 유도 | 히스토리 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| F008 | 업로드 로딩/에러 처리 | OCR 및 AI 처리 중 진행 상태 표시, 실패 시 재시도 안내 | 처리 시간이 5-15초로 길어 UX 필수 | 업로드 처리 페이지 |

## 개발 단계

전체 개발은 UI 검증 → 인터랙션 검증 → API 연동 → 완성 배포 순서로 진행한다. 각 단계는 이전 단계의 산출물을 기반으로 하며, 실제 API 비용과 복잡도를 단계적으로 분리하여 완성도를 높인다.

### Phase 0 - UI 프로토타입

- **목적**: 사용자 플로우 검증 및 디자인 확정
- **구현 방식**: 실제 기능 없이 화면 레이아웃과 반응형 구현 중심
- **산출물**: 4개 페이지 정적 화면
  - 홈 페이지
  - 업로드 처리 페이지 (로딩 화면)
  - 리뷰 결과 페이지
  - 히스토리 페이지

### Phase 1 - 더미 데이터 기반 개발

- **목적**: UX 및 인터랙션 사전 검증, 실제 API 연동 전 완성도 확보
- **구현 방식**: Mock 데이터로 화면 동작 구현, API 연결 없이 Zustand 상태 관리 중심 구성
- **산출물**: 실제처럼 동작하는 인터랙티브 프로토타입

### Phase 2 - 실제 API 연동

- **목적**: 실서비스 동작 구현
- **구현 방식**: 외부 서비스 순차 연동
  - Google Vision AI OCR 연동
  - Gemini API AI 리뷰 생성 연동
  - Supabase Storage/DB 연동

### Phase 3 - 완성 및 배포

- **목적**: 서비스 안정화 및 출시
- **구현 방식**: 에러 처리 및 엣지 케이스 보완, Vercel 배포

```

├── 홈
│   └── 기능: F001 (영수증 업로드 진입점)
├── 히스토리
│   └── 기능: F007, F005, F010 (과거 리뷰 조회/복사/삭제)

    └── 기능: F001 (카메라 직접 실행)

처리 흐름 페이지 (내비게이션 없음, 자동 전환)
```

## 페이지별 상세 기능

### 홈 페이지

- 영수증 이미지 파일 선택 (JPG/PNG/HEIC, 최대 10MB)
- 모바일 카메라 직접 실행 (capture="environment" 속성)
- 드래그 앤 드롭 업로드 (데스크탑)
- 서비스 소개 문구 및 사용 예시 3단계 안내
- **영수증 업로드** 버튼 (CTA)
- 이미지 선택 완료 → 업로드 처리 페이지로 자동 이동

### 업로드 처리 페이지

- 3단계 진행 상태 표시 (이미지 분석 중 → 정보 추출 중 → 리뷰 생성 중)
- 업로드한 영수증 이미지 썸네일 표시
- 예상 소요 시간 표시 (약 10-15초)
- 처리 실패 시 에러 메시지 + 재시도 버튼
- OCR 인식 불가 영수증 감지 시 안내 문구 표시

### 리뷰 결과 페이지

- OCR 추출 정보 카드 (상호명, 날짜, 주요 메뉴/상품, 금액) + 항목별 인라인 수정
- 리뷰 3버전 카드 탭 (짧게 / 보통 / 상세) - 탭 전환으로 확인
- 선택된 리뷰 텍스트 인라인 직접 수정 (textarea 활성화)
- **복사하기** 버튼 (원클릭, 복사 완료 시 토스트 메시지)
- **리뷰 다시 생성** 버튼 (Gemini 재요청, 로딩 스피너 표시)
- 리뷰 자동 저장 (로컬스토리지에 히스토리 추가)

### 히스토리 페이지

- 최근 리뷰 목록 카드 (상호명, 날짜, 리뷰 미리보기 첫 줄, 생성 일시 표시)
- 카드 탭으로 해당 리뷰 전문 확인 및 수정
- **복사하기** 버튼 (카드별 원클릭 복사)
- **삭제** 버튼 (카드별, 확인 다이얼로그 표시)
- 히스토리 없을 시 빈 상태 화면 + 영수증 업로드 유도 버튼

## 데이터 모델

### receipt_sessions (Supabase)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| image_url | String | Supabase Storage 영수증 이미지 경로 |
| ocr_raw | Text | Google Vision 원본 OCR 텍스트 |
| extracted_info | JSONB | 구조화된 OCR 결과 (상호명/메뉴/금액/날짜) |
| generated_reviews | JSONB | AI 생성 리뷰 3버전 배열 |
| created_at | Timestamp | 처리 일시 |

### review_history (LocalStorage)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID (nanoid) | 로컬 고유 식별자 |
| session_id | String | Supabase 세션 참조 |
| store_name | String | 상호명 |
| selected_review | Text | 사용자가 선택/수정한 최종 리뷰 텍스트 |
| review_type | String | 선택한 리뷰 버전 (short/medium/detail) |
| created_at | Timestamp | 저장 일시 |

## 핵심 UX 원칙

- 하단 내비게이션 바 고정 (엄지 접근성)
- 영수증 촬영 버튼은 하단 중앙 플로팅 버튼 (가장 큰 CTA)
- 리뷰 복사 버튼은 화면 하단 고정 (스크롤 없이 접근)
- 토스트 메시지는 하단 표시 (모바일 표준 위치)
- OCR + AI 처리 총 10-15초 소요 예상 → 3단계 진행 표시로 이탈율 감소
- 텍스트 에디터 활성화 시 키보드 올라와도 복사 버튼 접근 가능하도록 sticky 처리
