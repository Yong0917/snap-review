# 사진 리뷰 생성기 (SnapReview) MVP PRD

## 핵심 정보

**목적**: 음식, 공간, 제품, 소품 등 다양한 사진을 업로드하면 AI가 자연스러운 한국어 리뷰 문장을 자동 생성하여, 리뷰 작성에 드는 시간과 노력을 제거한다.  
**사용자**: 네이버/카카오/구글 등 플랫폼에 리뷰를 남기고 싶지만 문장 작성이 번거로운 사용자.

---

## 사용자 여정

```
1. 홈 페이지
   ↓ 사진 업로드 버튼 클릭 or 카메라 촬영 버튼 클릭

2. 업로드 처리 페이지
   ↓ 이미지 수신 → Gemini Vision 이미지 분석 → 리뷰 3버전 생성
   ↓ (처리 실패 시) 현재 페이지에 머물며 에러 메시지 + 재시도/홈 이동 선택

3. 리뷰 결과 페이지
   ↓ 이미지 분석 정보 확인 + AI 리뷰 3가지 버전 표시

   [리뷰 선택 후 그대로 복사] → 클립보드 복사 완료 → 홈 페이지 or 히스토리 페이지
   [리뷰 수정 원할 시] → 리뷰 에디터 활성화 → 수정 완료 → 클립보드 복사 완료
   [마음에 드는 버전 없을 시] → AI 재생성 요청 → 새 리뷰 3가지 표시

4. 히스토리 페이지
   ↓ 로컬스토리지에서 최근 리뷰 목록 표시

   [과거 리뷰 재사용] → 해당 리뷰 확인 → 수정 or 복사
   [히스토리 삭제] → 항목 삭제 확인 → 목록 갱신
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| F001 | 사진 업로드 | 카메라 촬영 또는 파일 선택으로 이미지 입력 | 서비스 진입점, 없으면 서비스 불가 | 홈 페이지 |
| F002 | 이미지 정보 추출 | Gemini Vision이 사진의 핵심 대상/분류/특징/분위기를 추출 | 리뷰 생성의 핵심 재료 확보 | 업로드 처리 페이지 |
| F003 | AI 리뷰 3버전 생성 | Gemini API로 짧게/보통/상세 3가지 리뷰 자동 생성 | 서비스의 핵심 가치 | 리뷰 결과 페이지 |
| F004 | 리뷰 인라인 에디터 | 생성된 리뷰를 직접 수정할 수 있는 텍스트 에디터 | 사용자 맞춤화 제공 | 리뷰 결과 페이지 |
| F005 | 원클릭 클립보드 복사 | 선택한 리뷰를 한 번의 탭/클릭으로 복사 | 서비스 완결점 | 리뷰 결과 페이지, 히스토리 페이지 |
| F006 | AI 리뷰 재생성 | 현재 리뷰가 마음에 들지 않을 때 새 버전 3가지 요청 | 사용자 만족도 보장 | 리뷰 결과 페이지 |
| F007 | 리뷰 히스토리 조회 | 로컬스토리지에 저장된 최근 리뷰 목록 표시 및 재사용 | 반복 사용 유도 | 히스토리 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| F008 | 업로드 로딩/에러 처리 | 이미지 분석 및 AI 처리 중 진행 상태 표시, 실패 시 재시도 안내 | 처리 시간이 길어 UX 필수 | 업로드 처리 페이지 |
| F009 | 이미지 정보 확인 | 추출된 대상명/분류/핵심 포인트/분위기를 사용자가 확인/수정 가능 | AI 해석 보정으로 리뷰 품질 향상 | 리뷰 결과 페이지 |
| F010 | 히스토리 항목 삭제 | 로컬스토리지에서 특정 히스토리 항목 삭제 | 개인 데이터 관리 기본 기능 | 히스토리 페이지 |

## 개발 단계

전체 개발은 UI 검증 → 인터랙션 검증 → API 연동 → 완성 배포 순서로 진행한다.

### Phase 0 - UI 프로토타입

- 목적: 사용자 플로우 검증 및 디자인 확정
- 산출물: 홈 / 처리 / 결과 / 히스토리 4개 정적 화면

### Phase 1 - 더미 데이터 기반 개발

- 목적: UX 및 인터랙션 사전 검증
- 산출물: 실제처럼 동작하는 인터랙티브 프로토타입

### Phase 2 - 실제 API 연동

- 목적: 실서비스 동작 구현
- 구현 방식:
  - Gemini Vision 이미지 분석 + 리뷰 생성
  - Supabase Storage/DB 연동

### Phase 3 - 완성 및 배포

- 목적: 에러 처리 및 엣지 케이스 보완, Vercel 배포

## 페이지별 상세 기능

### 홈 페이지

- 이미지 파일 선택 (JPG/PNG/HEIC, 최대 10MB)
- 모바일 카메라 직접 실행 (`capture="environment"`)
- 드래그 앤 드롭 업로드
- 서비스 소개 문구 및 사용 예시 3단계 안내
- 사진 선택 완료 시 업로드 처리 페이지로 자동 이동

### 업로드 처리 페이지

- 3단계 진행 상태 표시 (이미지 분석 중 → 핵심 포인트 추출 중 → 리뷰 생성 중)
- 처리 실패 시 같은 페이지에 에러 메시지 표시
- `다시 시도` / `홈으로 돌아가기` 버튼 제공

### 리뷰 결과 페이지

- 이미지 분석 정보 카드 (`subjectName`, `category`, `keyDetails`, `moodAndContext`) + 인라인 수정
- 리뷰 3버전 탭 (짧게 / 보통 / 상세)
- 선택된 리뷰 텍스트 직접 수정
- 복사 버튼
- 리뷰 다시 생성 버튼
- 복사 시 로컬스토리지 히스토리 자동 저장

### 히스토리 페이지

- 최근 리뷰 목록 카드 (제목, 분류, 리뷰 미리보기, 생성 일시 표시)
- 카드 탭으로 해당 리뷰 전문 확인
- 복사 버튼
- 삭제 버튼
- 히스토리 없을 시 빈 상태 화면 + 사진 업로드 유도 버튼

## 데이터 모델

### receipt_sessions (Supabase)

기존 테이블명을 유지하되, 저장 데이터는 일반 이미지 분석 기준으로 사용한다.

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| image_url | String | Supabase Storage 이미지 경로 |
| ocr_raw | Text | Gemini 원본 응답 텍스트 |
| extracted_info | JSONB | 구조화된 이미지 분석 결과 (`subjectName`, `category`, `keyDetails`, `moodAndContext`) |
| generated_reviews | JSONB | AI 생성 리뷰 3버전 |
| created_at | Timestamp | 처리 일시 |

### review_history (LocalStorage)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 로컬 고유 식별자 |
| session_id | String | Supabase 세션 참조 |
| title | String | 이미지 핵심 제목 |
| category | String | 이미지 분류 |
| reviews | JSON | 리뷰 3버전 |
| activeReview | String | 선택한 리뷰 버전 |
| createdAt | Timestamp | 저장 일시 |

## 핵심 UX 원칙

- 하단 내비게이션 바 고정
- 사진 업로드 버튼은 하단 중앙 플로팅 버튼
- 리뷰 복사 버튼은 화면 하단 고정
- 업로드 실패 시 자동으로 홈으로 돌려보내지 않고 현재 처리 화면에 머무르게 함
- AI 처리 시간 동안 3단계 진행 표시로 이탈율 감소
- 텍스트 에디터 활성화 시 모바일 키보드가 올라와도 복사 버튼 접근 가능해야 함
